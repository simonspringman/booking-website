Critical Issues — Fix Now

Critical rwandair.com + booking
Broken UI in production: currency selector shows "Please select a valid state"
A JavaScript validation error leaks into the default state of the currency dropdown before any user interaction. Every visitor sees a broken form element on page load.
Fix: Correct the default placeholder value in the select element. Low effort, high visibility.
Critical booking.rwandair.com
Post-payment silence: confirmed bookings show no charge for 24–72hrs with no explanation
Users receive a valid e-ticket but no card charge confirmation for days. The site offers no explanation anywhere — not on the confirmation page, not in email, not in the booking FAQ. This is the #1 driver of support calls and negative reviews on Trustpilot and FlyerTalk.
Fix: Add a single sentence to the confirmation screen: "Your payment is being processed and will appear on your statement within 72 hours."
Legal Both sites
GDPR-non-compliant cookie banner — no Reject or Manage option
The cookie banner reads "By browsing this website, you accept the use of cookies" with only an Accept button. No Reject or Manage Preferences option exists. Non-compliant for EU/UK markets (Brussels and London are key RwandAir routes).
Fix: Add Reject All and Manage Preferences buttons to the cookie banner.

Key Focus Areas

01
Payment anxiety & trust signals

No security badges at checkout. Fees hidden until the final step. Charge delay unexplained. #1 driver of negative reviews.

02
Booking widget decision paralysis

10+ tabs at equal visual weight mix booking actions with utility tools (Check-In, Flight Status). Needs hierarchy.

03
Error messages that teach, not punish

Generic "Please correct above error(s)" used across all failure states. No field-level guidance on what went wrong.

04
DreamMiles loyalty visibility

Loyalty portal routes to a third-party domain (dreammiles.frequentflyer.aero) without warning. No balance shown in booking flow.

05
Post-booking communication

No proactive delay alerts, no baggage tracking, no WhatsApp support. Users discover changes only at check-in.

06
Mobile & low-bandwidth UX

Two GTM containers, inline CSS/JS, and no effective lazy-loading on the destination grid. Critical for African mobile-first users.
