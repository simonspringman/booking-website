Architecture Overview

The middleware at wb-api.rwandair.com owns all Amadeus communication. The frontend barely changes.

Browser SPA
(index.html + app.js) → wb-api.rwandair.com
(Middleware / BFF) → Amadeus Digital APIs
(GDS / NDC)
wb-api.rwandair.com → Wallet Service    DreamMiles API    Payment Gateway    CMS / Admin DB

Frontend (Minimal Changes)

Swap fetch('/api/...') base URL to wb-api.rwandair.com. Keep same request/response shapes. Add auth token headers.

Middleware (Heavy Lift)

Transform Amadeus EDIFACT/JSON responses into our UI contract. Handle session management, caching, error translation, rate limits.

Internal Services (Keep As-Is)

Wallet, DreamMiles, CMS, Admin dashboard, chat assistant — these stay on your own infrastructure, no Amadeus dependency.
